# TurkiyeKala — خرید از ترکیه، تحویل در ایران

سایت فارسی و راست‌چین برای ثبت و مدیریت سفارش خرید کالا از ترکیه و تحویل در ایران.
ساخته‌شده با **Next.js App Router + TypeScript + Tailwind CSS + Supabase**.

مشتری فقط قیمت نهایی/حدودی تحویل تهران را می‌بیند؛ نرخ واقعی، سود چنج و جزئیات مالی
داخلی هرگز به مشتری نمایش داده نمی‌شود.

---

## سه مسیر ثبت سفارش

1. **Shopping** — انتخاب از پیشنهادهای تخفیف‌دار (`/shopping`)
2. **محصولات آماده** — محصولات تعریف‌شده در سایت (`/products`)
3. **سفارش با لینک** — ارسال لینک محصول از سایت‌های ترکیه‌ای (`/order-link`)

پیگیری سفارش با شماره سفارش یا موبایل در `/track`.

---

## پیش‌نیازها

- Node.js نسخه ۱۸ به بالا
- یک پروژه‌ی Supabase

## ۱) نصب

```bash
npm install
cp .env.example .env.local
```

سپس مقادیر `.env.local` را از داشبورد Supabase پر کنید:

```
NEXT_PUBLIC_SUPABASE_URL=...        # Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=...   # anon public key
SUPABASE_SERVICE_ROLE_KEY=...       # service_role key (فقط سمت سرور)
```

> کلید `SUPABASE_SERVICE_ROLE_KEY` نباید در کلاینت استفاده یا منتشر شود. این کلید فقط
> در Route Handlerها و توابع سرور (ثبت سفارش عمومی، پیگیری، خواندن نرخ‌ها) به‌کار می‌رود.

## ۲) ساخت دیتابیس

فایل‌های SQL داخل `supabase/migrations` را به ترتیب در **SQL Editor** داشبورد Supabase
اجرا کنید:

1. `0001_init.sql` — ساخت جداول، ENUMها، تریگرها، شماره سفارش و RLS
2. `0002_seed.sql` — تنظیمات پیش‌فرض + چند محصول و چند Shopping Product نمونه
3. `0003_create_admin.sql` — ساخت اولین ادمین (راهنمای زیر)

## ۳) ساخت اولین ادمین

1. در داشبورد Supabase → **Authentication → Users → Add user** یک کاربر با ایمیل و
   رمز بسازید و **Auto Confirm** را فعال کنید.
2. ایمیل همان کاربر را در `0003_create_admin.sql` بگذارید و اجرا کنید. این کاربر در
   جدول `admin_users` ثبت می‌شود.
3. وارد `/admin/login` شوید.

## ۴) اجرا

```bash
npm run dev      # حالت توسعه روی http://localhost:3000
npm run build    # بیلد پروداکشن
npm run start    # اجرای بیلد
npm run typecheck
```

## ۵) استقرار روی Vercel

1. ریپو را به Vercel وصل کنید.
2. سه متغیر محیطی بالا را در تنظیمات پروژه‌ی Vercel اضافه کنید.
3. Deploy.

## معماری قیمت‌گذاری

همه‌ی منطق در `lib/pricing.ts` متمرکز است:

```
مبلغ پرداختی مشتری =
  قیمت کالا (لیر) × نرخ تبدیل سایت
  + وزن × نرخ باربری هر کیلو × نرخ تبدیل سایت
  + کارمزد خدمات
```

- **کارمزد خدمات** = بزرگ‌ترِ (درصد کارمزد × مبلغ پایه) و (حداقل کارمزد)
- **وزن پیش‌فرض** بر اساس دسته‌بندی (اگر وزن وارد نشود): لباس سبک ۰٫۵، شلوار/هودی ۰٫۸،
  کفش ۱٫۲، کیف ۱٫۰، کاپشن ۱٫۸، لوازم خانه کوچک ۲٫۰ کیلو
- قیمت نمایشی به نزدیک‌ترین ۱۰۰۰ تومان گرد می‌شود.

مقادیر نرخ از جدول `settings` خوانده می‌شوند (نه هاردکد). پیش‌فرض‌ها:
نرخ باربری ۴۰۰ لیر/کیلو، کارمزد ۵٪، حداقل کارمزد ۲۵۰٬۰۰۰ تومان.

**نرخ واقعی/مرجع** (`real_exchange_rate_try_to_toman`) فقط برای گزارش سود داخلی در
پنل ادمین به‌کار می‌رود و در هیچ صفحه‌ی عمومی خوانده یا نمایش داده نمی‌شود.

## وارد کردن محصول از لینک

`lib/product-importer.ts` بدون هیچ browser automation (بدون Puppeteer/Playwright)
اطلاعات را از این منابع استخراج می‌کند:

1. Open Graph meta tags (`og:title`, `og:image`, `product:price:amount`, …)
2. JSON-LD Schema.org Product
3. fallback ساده‌ی HTML (`<title>`, canonical)

اگر دریافت خودکار موفق نشد، خطا نمی‌دهد و فرم دستی نمایش داده می‌شود. محصول ابتدا
به‌صورت **پیش‌نویس** ذخیره می‌شود و پس از بررسی ادمین در `/shopping` منتشر می‌گردد.

---

## امنیت (RLS)

- کاربران عمومی فقط محصولات `published` (Shopping) و محصولات غیرِ`inactive` را می‌بینند.
- کاربران عمومی فقط می‌توانند **سفارش ثبت** کنند (INSERT)، نه بخوانند.
- خواندن سفارش‌ها، نرخ‌ها، لاگ‌ها و مدیریت محصولات فقط برای ادمین (`is_admin()`).
- پیگیری عمومی از طریق API سمت سرور با نمایش محدود اطلاعات انجام می‌شود
  (نام مشتری ماسک، بدون آدرس/کدپستی).

---

## پرداخت (نسخه اول)

کارت‌به‌کارت + ثبت/آپلود لینک رسید + تأیید دستی توسط ادمین. درگاه پرداخت آنلاین در
فازهای بعدی اضافه می‌شود.

## ساختار وضعیت سفارش

۱۷ وضعیت از «در انتظار بررسی» تا «مبلغ بازگشت داده شد» در `lib/types.ts`
(`ORDER_STATUS_LABELS`). هر تغییر وضعیت به‌صورت خودکار توسط تریگر دیتابیس در
`order_status_logs` ثبت می‌شود.

## نقشه‌ی فایل‌ها

```
app/(public)/            صفحات عمومی (خانه، shopping، products، order-link، track، rules، faq، contact)
app/admin/login          ورود ادمین
app/admin/(panel)/       پنل مدیریت (داشبورد، سفارش‌ها، محصولات، shopping، import، settings)
app/api/                 orders، track، price/estimate، import
lib/pricing.ts           محاسبه قیمت و گزارش مالی
lib/product-importer.ts  استخراج اطلاعات از لینک
lib/types.ts             انواع و برچسب‌های وضعیت
lib/supabase/            کلاینت‌های browser / server / admin(service-role)
supabase/migrations/     SQL ساخت جداول + seed + ادمین
```
